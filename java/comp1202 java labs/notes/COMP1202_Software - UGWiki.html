[edit] The Java Development Kit

If you are using a iSolutions machine, then the latest version of the JDK is already installed.

If you are working on your own PC or laptop, the first step is to download the JDK.


[edit] Where is javac?

If you are running Windows, to be able to run javac (and other Java executables) from anywhere on your computer, you need to add the Java bin (short for binary) directory to the PATH environment variable. PATH is an Environment Variable. This is a globally accessible value that defines the list of directories to look in when a program is run without specifying the full path. It's a bit like having a list of places to look when you lose your glasses!

If you have this problem on your own computer you can fix it for good by adding your Java bin directory to your PATH variable.

Those of you running Linux don't have to worry about this, as you should be able to run the javac command from anywhere on your system by default.


[edit] On iSolutions Lab Machines

However, on the iSolutions computers you do not have permission to edit the PATH variable :-(

The solution is to temporarily set the PATH variable. The bad news is you will have to do this every time you open a command window.

The command to type is: set PATH=C:\Program Files\Java\jdk1.6.0_21\bin;%PATH%

However, to avoid having to type this every time, you can download this batch file which will perform this command for you. Just save it in the directory your code is in, and when you open your command window, cd to the code directory and type: 

C:\Users\abc123> javaPath.bat
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
| C:\Program Files\Java\jdk1.6.0_07\bin added to PATH |
|       Now type 'javac' to test the compiler         |
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

[edit] The Command Line

The command line interface is what existed before Englebart and his colleagues pioneered WIMP interfaces.

[edit] Windows

To get a command line window in Microsoft Windows:

  1. Click the Start button
  2. Click Run
  3. Type cmd
  4. Click OK

To get a better (Unix-like) command line window in Microsoft Windows:

  1. Download Cygwin
  2. Install Mintty from the installer
  3. Run Mintty

[edit] Mac

  1. Choose 'Finder' on the Dock
  2. Hit the "Go" button on the menu bar
  3. Click "Utilities"
  4. Double click "Terminal"

Mac OSX Terminal tutorial

[edit] Linux

If you're lucky and don't have a graphical user interface installed, then your Linux installation is only a command line! Hooray!

However, in all likelihood you will have a GUI, so will need to open a Terminal window. Try your applications menu or pressing Ctrl + Alt + T. (You might need to set this global shortcut before you can use it)

[edit] Text Editors

To write Java code, you just need to create a text file, type in your code and save it with the extension .java. Easy, huh?

Most operating systems come with a very basic text editor, and we recommend you use this to begin with so that you can see how simple it can be. After you are confident using a text editor and the command line, you can move onto more complicated development environments.

On Windows, the basic text editor is called Notepad. It can be found under the Accessories folder on the Start menu. Alternatively, you can click Run from the Start menu, type notepad and click OK.

On Linux, you should be able to run nano or pico from the command line, or find a text editor under the Applications menu. (Unabashed advert: why not try Geany it's gedit with more bells and whistles)

On Mac, look for TextEdit in spotlight, or in Applications.

[edit] Word Processors

Do not use a word processor such as Microsoft Word or OpenOffice Writer. They are for creating pretty looking documents, not for editing plain text. While you could use Save As... to save the file as .java, it's like using a sledgehammer or explosives to open a door.

[edit] Beyond Text Editors

Once you become familiar with the syntax of Java, you will find that you will find it very tiresome having to switch between windows to compile your code, checking if it works, and then switching back to fix error your program has just spat out at you. This is where an Integrated Development Environment (IDE) becomes useful. The three IDEs listed below are all compatible with Windows, Linux, Mac OSX and Solaris.

[edit] BlueJ

This is perhaps the first IDE you have come across. It's different from the more complicated IDEs, and has the benefit of being easy to use for beginners as classes are represented by images, and the relations between different classes can be seen. Delving deeper into BlueJ, a text editor with syntax highlighting can be found, which allows data types, variable names and Java's reserved words to be seen easier. Compiling and running your Java application can be done directly through BlueJ.

(Download BlueJ)

[edit] NetBeans

NetBeans is Syn Microsystems' (the people who developed the Java programming language) own IDE. Unlike BlueJ, all the programming must be done by hand; there are no visual aids showing the relationships between classes and you must write most of the code yourself. Without the visual aids, NetBeans is mainly a text editor, again with syntax highlighting however it also has a debugger which is very useful in finding out exactly what has gone wrong with your program. Using the debugger, you are able to stop your program running at a specific line in your code, and actually see what is stored in all the variables at the time of stopping.

Another feature to aid easier programming is syntax error highlighting and autocompletion. Syntax error highlighting is a feature which highlights (by underlining) the erroneous code, so that you can spot syntax errors before you compile the code. This makes life a lot easier as you don't have to spend hours looking for that missing semicolon! As you type, autocompletion will bring up a small drop-down list just below your caret (the blinking I, seen when typing) suggesting possible words you're about to type by looking at the first few letters you have typed.

(Download NetBeans)

[edit] Eclipse

Eclipse is generally the IDE most people will turn to despite being equivalent to NetBeans in most ways. In previous years, NetBeans really was a lot worse than Eclipse, but it has recently been improved and is now on par with Eclipse. Choosing between the two can be hard, but Eclipse has a steeper learning curve if you have never used an IDE before.

(Download Eclipse)
